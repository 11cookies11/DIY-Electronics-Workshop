# 前台 Agent 典型对话样例

## 样例 1：纯寒暄

用户：`你好呀`

预期：
- 走对话底座
- 不进入事务型 skill
- 不抢着收需求

## 样例 2：能力介绍

用户：`你能做什么？`

预期：
- 命中 `capability-intro`
- 先介绍能聊天、能梳理设备需求、能生成 3D 预览草案、能整理实验室交接单
- 不强行追问表单字段

## 样例 3：实验室介绍

用户：`介绍一下实验室`

预期：
- 命中 `lab-intro`
- 介绍实验室定位、常见设备方向和后续评估流程

## 样例 4：模糊需求澄清

用户：`我想做一个手持设备`

预期：
- 命中 `requirement-clarifier`
- 自然追问使用场景、核心功能、交互方式和供电方式
- 先问一个关键问题，不要连发很多问题

## 样例 5：手表方案推进到预览就绪

用户：`我想做一个带触摸屏和蓝牙的智能手表，电池供电，带 IMU 和 USB-C`

预期：
- 进入 `clarifying` 或 `preview_ready`
- 能给出 1 到 2 条常见建议
- 如果信息已足够，提示“可以准备 3D 草案”，但不要未确认就直接生成

## 样例 6：确认生成预览

上下文：上一轮前台已经说明“信息基本齐了，可以准备 3D 草案”

用户：`可以，来吧`

预期：
- 命中 `preview-promoter`
- 生成 `preview_input_draft`
- 状态推进到 `preview_generated`

## 样例 7：确认整理实验室交接单

上下文：上一轮已经生成预览草案

用户：`那就整理交接单吧`

预期：
- 命中 `handoff-promoter`
- 生成 `lab_handoff`
- 状态推进到 `handoff_ready`

## 样例 8：遥控器的合理追问

用户：`我想做一个万能遥控器，有触屏、红外、蓝牙和 Wi-Fi`

预期：
- 先指出这类设备通常还要确认供电方式和使用场景
- 能给建议，例如家用更适合充电电池或底座充电
- 在供电未确认前，不直接生成 3D 预览
